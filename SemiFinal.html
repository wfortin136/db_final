MPCS 53001 Web UI Using Ruby and CGI

List of Functions:

  • (1) List Tables: Displays all the tables and their row counts from the database
  • (2) List Fightcards: Displays all Fightcards and some general details of fightcard
    • (3) List Fights on Fightcard: Get the fights on a fightcard by selecting the fightcard from the Fightcard list (function 2)
    • (4) List Fight Stats: Get the fight stats of a fight selected from Fightcarddetails.html.erb (function 3)
    • (5) List Judgescores: Judgescores are combined with the fight stats in table in Fightdetails.html.erb
  • (6) List Fighters
    • (7) List round and total stats of fighter, selected from the list of fighters in fighters.html.erb
  • (8) List Judges
  • Search Database: Seach for a term across database. Functions will search across Fightcard Names, Fightcard Locations, Fighter Names and Judges Names.
    • (9) Search Fightcards
    • (10) Search Fighters
    • (11) Search Judges

    Example Search Terms:
    • "for": Return across all 3
    • "Fight": Return only on Fightcards
    • "Fahey": Return only on Fighters
    • "Chauncy": Return only on Judges

Description

I decided to implement my project with Ruby on Rails. After some testing on the linux server it became clear that the easiest way to deploy using RoR, would be to deploy on a server where I had root privelages. Prof. Freeman authorized this as long as I followed the same tree structure, /~/CS/mpcs53001/, which I have. As such, I am currently running my UI on my home server built from a Raspberry Pi. The full environment is as follows:

  • Hardware: Raspberry Pi 2 Model B
  • OS: Raspbian 7; Debian 7 (Wheezy)
  • HTTP Server: Apache 2.2.22 (Debian)
  • MySQL Server: MySQL 5.5.46-0+deb7u1 (Debian)
  • Ruby: Ruby 2.2.1p85 (2015-02-26 revision 49769)

Instead of deploying under the full rails framework, I have used Ruby in a very "non-rails" way by not leveraging the MVC rails framework. Instead, I have tried to mimic the deployment of php as closely as possible with Ruby.In order to do this, my server needed to convert html.erb files into CGI scripts. Once I was able to automatically convert embedded ruby into CGI commands, I could deploy *html.erb just like *.php files.

Additionally, I'm currently in the process of implementing the database within rails. This is under the RoR_version directory in the home diretory.This is not complete yet, but will provide a much cleaner implementation than the current html.erb files
